# agent-viz

Real-time pixel-art visualization dashboard for OpenClaw agent infrastructure.

## MVP Structure

- `bridge/` Node.js WebSocket bridge service with mocked OpenClaw state sources
- `frontend/` React + PixiJS v8 pixel office dashboard
- root workspace scripts to run both together

## Quick Start

```bash
npm install
npm run import:free
npm run dev
```

Local ports:
- frontend: `http://localhost:3000`
- bridge WebSocket: `ws://localhost:3001`

## Scripts

- `npm run dev` run bridge + frontend together
- `npm run dev:bridge` run bridge only
- `npm run dev:frontend` run frontend only
- `npm run build` build both workspaces
- `npm run import:free` import/normalize free OGA tilesets into frontend runtime atlas

## Free Asset Setup (OGA)

1. Place or refresh source files in:
- `assets/free-sources/opengameart/lab-office/`
- `assets/free-sources/opengameart/warped-tech-lab/`

2. Run importer:

```bash
npm run import:free
```

3. Generated runtime files:
- `frontend/public/assets/free/office.png`
- `frontend/public/assets/free/office.json`

For licensing and attribution details, see `ASSET_ATTRIBUTION.md`.

## Bridge Data Contract

Bridge pushes WebSocket messages in this form:

```ts
{
  type: "state",
  data: {
    agents: Array<{
      id: string;
      name: string;
      type: "persistent" | "contractor";
      status: "idle" | "thinking" | "working" | "error";
      connections: string[];
    }>;
    stats: {
      sessionCount: number;
      activeContractors: number;
      updatedAt: string;
    };
    events: Array<{ id: string; timestamp: string; message: string }>;
  }
}
```

## Notes

- OpenClaw gateway/session integrations are mocked for MVP but source classes are isolated for easy swap-in.
- The bridge still watches `~/.acpx/sessions/*.json` and emits mock file-change events.
- Pixel assets are generated by `frontend/scripts/generate_assets.py` into `frontend/public/assets/`.
